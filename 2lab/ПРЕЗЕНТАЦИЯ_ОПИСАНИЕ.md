# Описание функциональности системы "Cyber-Perceptron System Kernel v2.0"

## Общее описание
Кибернетическая платформа для моделирования нейросетевых процессов распознавания образов. Система реализует полный цикл работы однослойного персептрона: от формирования цифровых слепков (датасетов) до обучения нейроядра и идентификации паттернов в режиме реального времени. Интерфейс выполнен в стиле "Cyberpunk" для максимального погружения в процесс проектирования интеллектуальных систем будущего.

---

## МОДУЛЬ 1: "Обучающая выборка" (Dataset Protocol)

### Назначение
Формирование базы данных сигнатур (изображений) для тренировки нейроядра. Каждая сигнатура должна быть маркирована уникальным идентификатором класса.

### Элементы интерфейса и их функции:

#### 1. Форма загрузки сигнатур (Input Terminal)

**Поле "Метка класса"** (`sample-label`)
- **Функция:** Присвоение текстового идентификатора для паттерна.
- **Примеры:** "ЦЕЛЬ_ЗАХВАТ", "ЗАХВАТЧИК", "БЛОК_ДАННЫХ".
- **Статус:** Обязательно для заполнения.

**Поле "Чёрно-белое изображение"** (`sample-file`)
- **Функция:** Загрузка графического файла для анализа.
- **Формат:** Любой растровый формат (JPG, PNG и др.).
- **Обработка:** Автоматическая бинаризация и ресайзинг.

**Поля "Ширина" и "Высота"** (`image-width`, `image-height`)
- **Функция:** Определение размерности входной матрицы нейросети.
- **Диапазон:** 3-12 пикселей.
- **Стандарт:** 8×8 пикселей.
- **Внимание:** Все паттерны в одной сессии должны иметь идентичную размерность.

**Поле "Порог бинаризации"** (`threshold`)
- **Функция:** Калибровка чувствительности сенсора (0-255).
- **Значение по умолчанию:** 128.
- **Логика:** Значение > порога → 1 (сигнал), иначе → 0 (шум).

#### 2. Командные кнопки (Control Unit)

**Кнопка "Добавить в выборку"** (Execute Add)
- **Алгоритм:**
  1. Считывание файла.
  2. Преобразование в матрицу пикселей (0/1).
  3. Маркировка классом.
  4. Сохранение в `cyber-perceptron-dataset`.
  5. Визуализация в реестре данных.

**Кнопка "Загрузить демо-набор (12)"** (Load Cyber-Patterns)
- **Функция:** Инициализация стандартного набора кибер-паттернов:
  - **ЦЕЛЬ_ЗАХВАТ, ЗАХВАТЧИК**
  - **БЛОК_ДАННЫХ, СИГНАЛ, ЯДРО_CPU**
  - **ЦИФРА_0, ЦИФРА_1, МЕТКА_X**
  - **ПЛЮС, УГОЛ_L, СБОЙ, СТРЕЛКА_ВВЕРХ**
- **Назначение:** Быстрое развертывание тестового окружения.

**Кнопка "Очистить"** (System Purge)
- **Функция:** Полный сброс памяти системы, включая обученные веса и историю экспериментов.

#### 3. Панель "Состояние выборки" (Data Registry)
- **Индикация:** Общее число паттернов и уникальных классов.
- **Визуализация:** Матрица миниатюр загруженных образов с возможностью точечного удаления (УДАЛИТЬ).

## МОДУЛЬ 2: "Обучение" (Training Sequence)

### Назначение
Запуск алгоритма обратного распространения ошибки (в упрощенном виде дельта-правила) для настройки синаптических весов персептрона.

### Параметры обучения (Hyperparameters):

**Скорость обучения η** (`learning-rate`)
- **Описание:** Коэффициент адаптации весов (0.01 - 1.0).
- **Рекомендация:** 0.2 для баланса скорости и стабильности.

**Максимум эпох** (`max-epochs`)
- **Описание:** Предел итераций обучения (защита от зацикливания).

**Порог ошибки ε (MSE)** (`target-error`)
- **Описание:** Целевая точность. Обучение прерывается при достижении MSE < ε.

**Инициализация весов (±)** (`weight-range`)
- **Описание:** Стартовый разброс значений синапсов (рандомизация).

**Активационный порог θ** (`activation-threshold`)
- **Описание:** Уровень сигнала, необходимый для активации нейрона.

### Визуализация процесса (Training Logs):

**Протокол обучения**
- Таблица, отображающая ход обучения в реальном времени (первые 100 итераций).
- Поля: Эпоха, Образец, Нейрон, Net (сумма), Ошибка, ΔW (изменение весов).

**Веса и пороги нейронов**
- Матрица финальных весов для каждого класса после завершения обучения.
- Возможность экспорта весов в JSON (Скопировать JSON).

**График ошибки (MSE Chart)**
- Неоновый график сходимости алгоритма. Показывает динамику уменьшения ошибки от эпохи к эпохе.

## МОДУЛЬ 3: "Распознавание" (Pattern Recognition)

### Назначение
Использование обученного нейроядра для классификации входящих сигналов.

### Режимы работы:

1.  **Тест на выборке:** Выбор известного паттерна из базы данных для проверки "памяти" системы.
2.  **Скан нового файла:** Загрузка внешнего изображения для проверки способности к обобщению.

### Результаты сканирования:
- **Идентифицированный класс:** Вердикт системы.
- **Детализация:** Таблица значений NET (активность нейронов) для каждого класса. Побеждает класс с максимальным NET.

## ДОПОЛНИТЕЛЬНЫЕ МОДУЛИ

### Протоколы испытаний (Experiment Logs)
- Автоматический журнал последних 12 запусков обучения.
- Фиксирует: Время, Размер базы, MSE, Точность (Accuracy).
- Поле для заметок инженера (Notes) с автосохранением.

### Системный статус (System Status)
- Индикатор в футере: "СИСТЕМА В СЕТИ".
- Подтверждение работоспособности ядра.

---

## ТЕХНИЧЕСКАЯ СПЕЦИФИКАЦИЯ

### Хранилище данных (Storage Core)
- **Технология:** LocalStorage.
- **Ключи доступа:**
  - `cyber-perceptron-dataset`: База сигнатур.
  - `cyber-perceptron-experiments`: Журнал испытаний.
  - `cyber-perceptron-notes`: Заметки оператора.

### Алгоритмическое ядро (Kernel Logic)
- **Метод обучения:** Дельта-правило (Delta Rule).
- **Функция активации:** Пороговая (Heaviside Step Function).
- **Векторизация:** Преобразование изображений в бинарные векторы фиксированной длины.

### Требования к системе
- Браузер с поддержкой ES6+ и Canvas API.
- Разрешение экрана: мин. 1024px по ширине (рекомендуется FHD).
